#### Служебные пользователи

В этом разделе вы узнаете:

* [Как подключить сервисную учетную запись Пользователь API](https://wiki.hubex.ru/docs/FAQ/RU/admin/ServiceUsers.html.txt#API);
* [Как подключить сервисную учетную запись Анонимного пользователя](https://wiki.hubex.ru/docs/FAQ/RU/admin/ServiceUsers.html.txt#Anonym).

Служебные пользователи или сервисные учетные записи необходимы для интеграции с внешними системами (Битрикс24),
электронной почтой и
подачи заявок незарегистрированными пользователями. Подробнее читайте в статье: [Интеграция с Битрикс24 через сервис Albato](https://wiki.hubex.ru/docs/FAQ/RU/admin/Integration.html.txt),
[Интеграция по электронной почте, подача
заявок по электронной почте](https://wiki.hubex.ru/docs/FAQ/RU/admin/TicketMail.html.txt),
[Подача заявок по QR-коду без
приложения, самостоятельная регистрация нового пользователя](https://wiki.hubex.ru/docs/FAQ/RU/user/SelfRegister.html.txt).

Подключить и настроить сервисные учетные записи можно в [консоли
администратора](https://wiki.hubex.ru/docs/FAQ/RU/admin/HowToEnterTheAdmin.html.txt) в меню **Интеграция - Служебные пользователи**.

##### Подключение сервисной учетной записи Пользователь API

**Пользователь API** используется для настройки интеграции со сторонними системами (Битрикс24) и
электронной почтой.

В рамках данной статьи будут даны полные настройки для **Пользователя API**.

Переключите тумблер
вправо и установите необходимые настройки:

* выберите **Роли**, которые наделены максимальными полномочиями для работы с
  **Заявками** и
  **Объектами**. При интеграции с выбранных для **Пользователя API** ролей будут
  считываться полномочия. Обычно
  полными правами наделены Администраторы, Системные администраторы, Руководители служб.
* также укажите **Участки**, для которых будет доступна интеграция. Рекомендуем выбрать
  все участки. Подробнее об участках читайте в статье: [Настройка
  участков](https://wiki.hubex.ru/docs/FAQ/RU/admin/Places.html.txt).
* в поле **Токен доступа** можно сгенерировать специальный ключ, по которому сторонняя система будет
  определять пользователя HubEx. Для этого нажмите кнопку **Сгенерировать** и скачайте сформированный
  токен доступа на компьютер.

В скачанном текстовом документе будет находиться ключ, который пригодится для интеграции с внешними системами,
такими как Битрикс24. Для интеграции с почтой токен гененировать не нужно.

![](https://wiki.hubex.ru/attachments/images/FAQ/ADMIN/ServiceUsers/ServiceUsers.jpg)

##### Подключение сервисной учетной записи Анонимный пользователь

**Анонимный пользователь** используется для того, чтобы незарегистрированные пользователи могли
подавать **Заявки** с помощью QR-кодов.

В настройках **Анонимного пользователя** укажите:

* какая **Роль** будет использована для незарегистрированного пользователя. Например,
  **Заказчик**. Если анонимный пользователь захочет пройти процедуру регистрации в HubEx, ему будет присвоена указанная роль.
* Укажите **Участки**, по которым возможно создание
  **Заявок**. Вы можете выбрать все участки. Подробнее читайте в статье: [Настройка участков](https://wiki.hubex.ru/docs/FAQ/RU/admin/Places.html.txt).
