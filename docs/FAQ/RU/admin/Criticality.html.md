#### Критичность заявки

**Критичность заявки** позволяет:

* разделять **Заявки** по важности и срочности. Критичность отображается в списке **Заявок** web- и
  мобильных приложений цветным маркером и помогает вашим специалистам
  понимать приоритет выполнения задач;
* настраивать правила расчета крайнего срока закрытия заявки. По очень срочным **Заявкам** срок закрытия может быть гораздо меньше, чем по стандартным (несрочным). Подробнее читайте в статье: [SLA - Правила расчета
  крайнего срока закрытия заявки](https://wiki.hubex.ru/docs/FAQ/RU/admin/SLA.html.md).

![](https://wiki.hubex.ru/attachments/images/FAQ/ADMIN/Criticality/Ticklist.jpg)

Создать новую критичность можно в [консоли
администратора](https://wiki.hubex.ru/docs/FAQ/RU/admin/HowToEnterTheAdmin.html.md) в меню **Настройки заявки - Критичность заявки** по кнопке
**Создать критичность заявки**.

![](https://wiki.hubex.ru/attachments/images/FAQ/ADMIN/Criticality/critic1.png)

В форме создания критичности введите название и выберите цвет, который позволит вам определять приоритет **Заявок**.
Установленный флажок **Для всех заявок по умолчанию** означает, что данная критичность будет устанавливаться
автоматически при создании новой **Заявки**.

![](https://wiki.hubex.ru/attachments/images/FAQ/ADMIN/Criticality/critic2.png)
