#### SLA - Правила расчета крайнего срока закрытия заявки

Важный критерий качества предоставляемых услуг - соблюдение сроков выполнения заявок. Временные нормы могут быть
прописаны в
договорах (контрактах) с вашими заказчиками и отличаться по критичности, видам выполняемых работ и т.д.

Закрепленные в договорах показатели качества услуг называют специальным термином - SLA (Service Level Agreement,
Соглашение об уровне сервиса).

Прочтите подробную статью ниже или начните знакомство с темой с обучающего видеоролика **Настройка SLA: расчет крайнего срока закрытия заявки**.

##### Правила расчета крайнего срока закрытия заявки

Учет SLA в HubEx позволяет создавать гибкие правила для расчета крайнего срока закрытия заявки. При совпадении
параметров в **Заявке** и правиле расчета **Крайний срок закрытия** (поле в **Заявке**) заполняется автоматически. Правило, по
которому был рассчитан срок будет указано на форме **Заявки** внизу.

![](https://wiki.hubex.ru/attachments/images/FAQ/ADMIN/SLA/Ticket.jpg)

Если **Крайний
срок закрытия** в **Заявке** был установлен вручную, он не будет изменен с помощью правила. Если в системе не настроено или
не включено ни одного правила срок закрытия может быть установлен по параметрам, заданным в **Типе заявки** или **Виде
работ**. Подробнее читайте в статьях: [Типы заявок](https://wiki.hubex.ru/docs/FAQ/RU/admin/TicketType.html.txt), [Виды работ](https://wiki.hubex.ru/docs/FAQ/RU/admin/WorkType.html.txt).

Правила расчета крайнего срока закрытия находятся в консоли администратора в меню **Настройки заявки - SLA: расчет
крайнего срока закрытия**.

![](https://wiki.hubex.ru/attachments/images/FAQ/ADMIN/SLA/SLA.jpg)

Создать новое можно по кнопке **Создать правило**.

![](https://wiki.hubex.ru/attachments/images/FAQ/ADMIN/SLA/SLA2.jpg)

В названии введите наименование правила для удобной идентификации. Название правила будет отображаться в поле **Правило расчета крайнего срока закрытия** на форме **Заявки**.

Правило расчета времени закрытия заявки может быть настроено в разрезе **Типа заявки**, **Критичности**, **Вида работ**, а также
индивидуально для конкретной **Компании-заказчика(ов)** и **Объекта(ов)** обслуживания. При создании **Заявки** система определит
соответствие с параметрами, указанными в правилах и установит значение даты и времени в поле **Крайний срок закрытия** в
**Заявке**.

Правило можно создать только на один **Тип заявки**, одну **Критичность** и один **Вид работ**. Либо по каждому из пунктов можно
выбрать значение **Все** - оно
означает, что правило будет работать для всех **Типов заявок**, **Видов работ** и **Всех критичностей**.

**Заказчика(ов)** можно выбрать из справочника **Компании** с помощью следующих параметров:

* **Все**: означает, что правило расчета крайнего срока будет действовать на всех уже внесенных в систему **Компаний-заказчиков**,
  а также на тех, которые будут создаваться в будущем;
* **По списку**: позволяет выбрать из списка уже созданных заказчиков.

![](https://wiki.hubex.ru/attachments/images/FAQ/ADMIN/SLA/SLA3.jpg)

При выборе **Заказчика(ов)** становится доступным выбор **Объектов**, которые принадлежат выбранному
**Заказчику(ам)**.

**Объекты** можно выбрать по следующим параметрам:

* **Все**: означает, что правило расчета крайнего срока будет действовать на все уже внесенные в систему **Объекты**
  выбранного **Заказчика**, а также на те, которые будут создаваться в будущем;
* **По списку**: позволяет выбрать из списка уже созданных **Объектов**;

![](https://wiki.hubex.ru/attachments/images/FAQ/ADMIN/SLA/SLA4.jpg)

* **Все за исключением**: правило будет распространяться на все **Объекты**, которые НЕ выбраны в исключениях, а
  также на те, которые будут создаваться в будущем.

![](https://wiki.hubex.ru/attachments/images/FAQ/ADMIN/SLA/SLA5.jpg)

**График обслуживания** определяет рабочее время выполнения **Заявок**. **Графики обслуживания**, как и **Графики работ**
исполнителей встроены в систему. Если вы не нашли подходящий,
напишите в Чат с
поддержкой в web-приложении или на почту [help@hubex.ru](mailto:help@hubex.ru). Мы добавим ваш уникальный график.

**Плановое время закрытия заявки** определяет максимальный срок выполнения работ по **Заявке**.
Например, обслуживание заказчика происходит по будням с 9:00 до 18:00, **Заявка** была подана в четверг в 17.00,
**Плановое время закрытия заявки** равно 6 часам, значит **Крайний срок закрытия** будет установлен на пятницу
14:00.

Чтобы правило начало работать, его необходимо включить. Включить правило также можно позже в общем списке всех правил
с помощью переключателя (установить в положение "Включено").

![](https://wiki.hubex.ru/attachments/images/FAQ/ADMIN/SLA/SLA6.jpg)

Если в разрезе **Типа заявки**, **Критичности**, **Вида работ**, **Заказчика** и **Объекта** будет создано несколько правил, работать
будет, то, которое по **Графику обслуживания** попадает на текущее время. Если совпадают **Графики обслуживания**, то
выбрано будет правило с наименьшим сроком закрытия.
