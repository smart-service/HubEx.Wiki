<p>В эпоху цифровых технологий организации постоянно ищут способы оптимизировать внутренние процессы, максимизировать
    производительность и улучшить взаимодействие между сотрудниками. Одним из мощных инструментов является корпоративная
    база знаний.</p>

<p>В HubEx база знаний является дополнительно подключаемым виджетом компании-партнера HiHub.</p>
<p>Узнать подробность и подключить базу знаний в HubEx можно по запросу нашу службу поддержки на почту <a
        href="mailto:support@hubex.ru"
        target="_blank" rel="noopener">support@hubex.ru</a>.
</p>

<p>После подключения виджета для вашего аккаунта в меню HubEx появится отдельный пункт <strong>База знаний</strong>.</p>
<p>Обратите внимание! Виджет доступен пользователям, для которых в настройках ролей включено полномочие <strong>Доступ к подключаемым плагинам и отчетам</strong>.</p>

<h5 id="start">Начало работы с базой знаний</h5>
<p>При первом взаимодействии с базой знаний необходима авторизация в сервисе Hihub. В сервисе необходимо
    зарегистрировать свой адрес электронной почты по кнопке
    <strong>Регистрация</strong>. </p>
<p>Наша команда рекомендует проходить регистрацию в базе знаний пользователю, который является администратором
    бизнес-аккаунта HubEx. Такой пользователь будет являться создателем для базы знаний Hihub.
    И затем создать несколько дополнительных аккаунтов для пользователей с разным уровнем доступа. Об этом читайте в
    разделе этой статьи <a
            href="#users">Как предоставить доступ пользователям</a>.</p>

<h5 id="branch">Создание рабочих пространств</h5>
<p>Для разных отделов или проектов в базе знаний можно создавать разные рабочие пространства. Разные пространства
    помогут сотрудникам легко ориентироваться в статьях. </p>
<p>Создать новое пространство можно в основном меню в разделе <strong>Пространства</strong> по кнопке <strong>Создать новое рабочее
    пространство</strong>.</p>

<div>

<p>При создании нового пространства введите его название и описание.</p>

<h5 id="article">Создание статей и разделов</h5>
<p>Разделы - это группировка статей внутри одного пространства. Разделы позволят разделить статьи по различным
    критериям, тем самым обеспечить пользователям убобный поиск и работу со статьями.</p>
<p>Создать новый раздел можно на основной странице по кнопке <strong>+Добавить статью (кнопка открытия выпадающего списка \/) -
    Добавить раздел</strong>. </p>

<p>При создании раздела нужно ввести наименование и при необходимости заполнить описание и добавить картинку для
    визуального отображения в базе знаний. Сохранить новый раздел можно по кнопке внизу страницы <strong>Сохранить</strong>.</p>

<h5 id="users">Пользователи для базы знаний</h5>
<p>Для работы в базе знаний вы можете зарегистрировать 5 учетных записей (включая ту, которую создает администратор
    аккаунта при первом входе в базу знаний).</p>
<p>Администратор может зарегистрировать несколько учетных записей и выдать логин и пароль всем сотрудникам для входа в
    базу знаний.</p>
<p>Обратите внимание! Если вы создали несколько пространств, то сначала необходимо выбрать пространство и затем
    создавать для него учетную запись. Если вы хотите создать учетную запись для доступа ко всем пространствам, то
    выберите в основном меню пространство <strong>Общая база знаний</strong>.</p>

<p>Зарегистрировать новую учетную запись можно в основном меню <strong>Инструменты - Приглашения</strong>. </p>
<div>
        <img style="margin: 0 auto; display: block; max-width: 95%;"
             src="/attachments/images/FAQ/USER/KnowledgeBase/Invite.jpg"/>
    </div>

<p>При создании учетных записей, одна из них, например, может быть только для чтения статей. Логин и пароль от этой
    записи вы можете передать своим сотрудникам для входа в базу знаний. Несколько сотрудников будут работать под одной
    учетной записью именно в базе знаний.</p>

<p>При регистрации новой записи необходимо указать электронную почту и можно заполнить Имя или Фамилию. Например, можно
    создать отдельную учетную запись для отдела сервиса, чтобы они просматривали инструкции по работе с объектами и
    оборудованием. </p>

<div>
    <img style="margin: 0 auto; display: block; max-width: 95%;"
         src="/attachments/images/FAQ/USER/KnowledgeBase/NewUser.jpg"/>
</div>

<p>После создания учетной записи можно настроить уровень доступа для нее.</p>
<p>В базе знаний есть три уровня доступа пользователей (три роли):</p>
<ul>
    <li><strong>Администратор</strong>: администратор помимо всех действий со статьями, может приглашать пользователей, управлять ролями,
        настраивать доступ к статьям,
        просматривать историю версий статей, управлять обязательностью прочтения. Создатель базы знаний (пользователь,
        который первым регистрируется в базе знаний) по умолчанию является администратором;
    </li>
    <li><strong>Редактор</strong>: этой роли доступно редактирование статей, создание новых статей, просмотр архива и черновиков, а также
        возможность установить доступ к разделам;
    </li>
    <li><strong>Пользователь</strong>: пользователю доступен просмотр статей, добавление их в избранное. Пользователь не может добавлять
        или удалять статьи в базе знаний.
    </li>
</ul>
<p>Установить роль для учетной записи можно в основном меню <strong>Инструменты - Пользователи пространства</strong>.</p>
