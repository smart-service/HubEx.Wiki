<h3><strong><span class=""><br />🔍 Настройка полей поиска объектов: теперь вы выбираете, по чему искать</span></strong></h3>
<p class="ds-markdown-paragraph"><span class="">Мы добавили возможность гибко настраивать поля, по которым выполняется поиск объектов. Теперь вы можете выбрать только релевантные поля, чтобы поиск был точнее и быстрее, а результаты &mdash; полезнее.</span></p>
<h4><strong><span class="">Что изменилось?</span></strong></h4>
<p class="ds-markdown-paragraph"><strong><span class="">Раньше:</span></strong><span class="">&nbsp;поиск всегда выполнялся по всем полям объекта одновременно, без возможности повлиять на этот набор.</span></p>
<p class="ds-markdown-paragraph"><strong><span class="">Теперь:</span></strong><span class="">&nbsp;рядом со строкой поиска появилась кнопка, которая открывает окно настройки&nbsp;</span><strong><span class="">полей поиска</span></strong><span class="">. В этом окне вы можете отметить чекбоксами те поля, по которым хотите искать объекты.</span></p>
<p><img style="max-width: 80%; max-height: 1000px; height: auto; display: block; margin-left: auto; margin-right: auto;" src="https://wiki.hubex.ru/attachments/images/FAQ/RELEASENOTES/ListOfChanges20072026/IMAGE4.jpg" alt="" /></p>
<h4><strong><span class="">Как это работает</span></strong></h4>
<ol start="1">
<li>
<p class="ds-markdown-paragraph"><span class="">Нажмите на кнопку настройки полей поиска.</span></p>
</li>
<li>
<p class="ds-markdown-paragraph"><span class="">В открывшемся списке отображаются все доступные для поиска поля (например,&nbsp;</span><strong><span class="">Название</span></strong><span class="">,&nbsp;</span><strong><span class="">Серийный номер</span></strong><span class="">,&nbsp;</span><strong><span class="">Адрес</span></strong><span class="">).</span></p>
<ul>
<li>
<p class="ds-markdown-paragraph"><span class="">Список формируется автоматически с учётом ограничений вашего тенанта.</span></p>
</li>
<li>
<p class="ds-markdown-paragraph"><span class="">По умолчанию могут быть выбраны все поля (или те, которые вы сохранили ранее).</span></p>
</li>
</ul>
</li>
<li>
<p class="ds-markdown-paragraph"><span class="">Отметьте нужные поля и нажмите&nbsp;</span><strong><span class="">&laquo;Сохранить&raquo;</span></strong><span class="">.</span></p>
</li>
<li>
<p class="ds-markdown-paragraph"><span class="">Теперь при вводе запроса в строку поиска система будет искать&nbsp;</span><strong><span class="">только по выбранным полям</span></strong><span class="">.</span></p>
</li>
</ol>
<p><img style="max-width: 80%; max-height: 1000px; height: auto; display: block; margin-left: auto; margin-right: auto;" src="https://wiki.hubex.ru/attachments/images/FAQ/RELEASENOTES/ListOfChanges20072026/IMAGE5.jpg" alt="" /></p>
<h4><strong><span class="">Важные правила использования</span></strong></h4>
<ul>
<li>
<p class="ds-markdown-paragraph"><span class="">Чтобы начать поиск, нажмите&nbsp;</span><code>Enter</code><span class="">&nbsp;или кнопку&nbsp;</span><strong><span class="">&laquo;Найти&raquo;</span></strong><span class="">.</span></p>
</li>
<li>
<p class="ds-markdown-paragraph"><strong><span class="">Автоматический поиск по таймауту (без нажатия) отключён</span></strong><span class="">&nbsp;&mdash; искать нужно только явным действием.</span></p>
</li>
<li>
<p class="ds-markdown-paragraph"><span class="">Если в настройках выбрано&nbsp;</span><strong><span class="">ровно одно поле</span></strong><span class="">, множественный поиск работает в обычном режиме.</span></p>
</li>
<li>
<p class="ds-markdown-paragraph"><span class="">Если выбрано</span><strong><span class="">&nbsp;больше одного поля</span></strong><span class="">, множественный поиск недоступен, а при попытке его запустить появится понятная подсказка.</span></p>
</li>
</ul>
<h4><strong><span class="">Преимущества для пользователей</span></strong></h4>
<ul>
<li>
<p class="ds-markdown-paragraph"><strong><span class="">Релевантные результаты</span></strong><span class="">&nbsp;&mdash; вы ищете только по тем полям, которые важны для вашей задачи.</span></p>
</li>
<li>
<p class="ds-markdown-paragraph"><strong><span class="">Экономия времени</span></strong><span class="">&nbsp;&mdash; не нужно просматривать лишние записи, подходящие по другим полям.</span></p>
</li>
<li>
<p class="ds-markdown-paragraph"><strong><span class="">Персонализация</span></strong><span class="">&nbsp;&mdash; настройки сохраняются и автоматически применяются при следующих входах.</span></p>
</li>
<li>
<p class="ds-markdown-paragraph"><strong><span class="">Прозрачность</span></strong><span class="">&nbsp;&mdash; вы всегда видите, по каким полям сейчас выполняется поиск.</span></p>
</li>
</ul>

<h3><strong><span class=""><span data-teams="true">🚗 Экран пробегов сотрудников в Истории перемещений</span></span></strong></h3>
<p>Мы расширили функционал &laquo;Истории перемещений&raquo; &mdash; теперь вы можете не только видеть маршруты сотрудников на карте, но и анализировать их километраж за день. Новый раздел помогает быстро оценить дневной пробег каждого исполнителя и сравнить показатели внутри команды.</p>
<h4>Что нового?</h4>
<p><strong>1. Маршрут в боковой панели</strong><br /> При выборе сотрудника и даты в боковой панели теперь отображается общий пробег за выбранный день. Данные рассчитываются автоматически на основе точек маршрута &mdash; вы сразу видите итоговое расстояние без дополнительных действий.</p>
<p><img style="max-width: 80%; max-height: 1000px; height: auto; display: block; margin-left: auto; margin-right: auto;" src="https://wiki.hubex.ru/attachments/images/FAQ/RELEASENOTES/ListOfChanges20072026/IMAGE6.jpg" alt="" /></p>
<p><strong>2. Экран &laquo;Пробег&raquo;</strong><br /> Новый табличный вид показывает сводку пробегов всех мобильных сотрудников за выбранную дату:</p>
<ul>
<li>Список сотрудников с указанием дневного километража</li>
<li>Компактная таблица для быстрого сравнения показателей команды</li>
<li>Удобно для формирования отчётности и контроля использования транспорта</li>
</ul>
<p><img style="max-width: 80%; max-height: 1000px; height: auto; display: block; margin-left: auto; margin-right: auto;" src="https://wiki.hubex.ru/attachments/images/FAQ/RELEASENOTES/ListOfChanges20072026/IMAGE7.jpg" alt="" /></p>
<p><strong>3. Быстрый переход к маршруту</strong><br /> Клик по строке сотрудника в таблице автоматически переключает вас на карту и загружает маршрут выбранного исполнителя. Больше не нужно вручную искать сотрудника в выпадающем списке &mdash; данные и визуализация в одном клике.</p>
<h4>Важно</h4>
<ul>
<li>Пробег рассчитывается на основе данных GPS-трекинга мобильных сотрудников. Для получения корректных данных у сотрудника должна быть включена функция &laquo;Отслеживать местоположение&raquo;.</li>
</ul>
<h4>💬 Улучшены всплывающие подсказки на карте</h4>
<p>Мы доработали тултипы для объектов и заявок на карте &laquo;Истории перемещений&raquo;, чтобы длинные адреса и названия отображались корректно.</p>
<h4>Что изменилось?</h4>
<ul>
<li><strong>Увеличили ширину всплывающих подсказок</strong> &mdash; теперь даже длинные адреса и названия объектов помещаются без обрезки.</li>
<li><strong>Перенос текста по словам</strong> &mdash; текст больше не обрывается посередине слова. Перенос выполняется по правилам русского языка, что делает подсказки чище и читабельнее.</li>
</ul>
<p>Новый функционал уже доступен в разделе &laquo;Карты&raquo; &rarr; &laquo;История перемещений&raquo;. Попробуйте проанализировать пробег сотрудников и оцените обновлённые подсказки на карте!</p>

<h3><strong><span class="">🖱️ Панель быстрых действий в списках заявок, объектов, складов и материалах теперь не сдвигает страницу</span></strong></h3>
<p class="ds-markdown-paragraph"><span class="">Мы устранили досадный визуальный эффект при массовом выборе заявок. Теперь панель быстрых действий появляется и скрывается плавно, без &laquo;прыжка&raquo;&nbsp;страницы вниз или вверх.</span></p>
<h4><strong><span class="">В чём была проблема?</span></strong></h4>
<p class="ds-markdown-paragraph"><span class="">При выборе заявок через чек-боксы в верхней части страницы появлялась панель с кнопками массовых действий (удалить, изменить и т.д.). Это происходило с анимацией, которая&nbsp;</span><strong><span class="">сдвигала весь список заявок вниз</span></strong><span class="">, создавая резкий визуальный скачок. Особенно это мешало, когда пользователь находился в середине или в конце списка &mdash; после появления панели он &laquo;терял&raquo; текущее место.</span></p>
<h4><strong><span class="">Что изменилось?</span></strong></h4>
<ul>
<li>
<p class="ds-markdown-paragraph"><span class="">Панель быстрых действий&nbsp;</span><strong><span class="">появляется на своём месте</span></strong><span class="">, но список заявок остаётся строго на той же позиции, где был.</span></p>
</li>
<li>
<p class="ds-markdown-paragraph"><span class="">При выборе первой заявки и при снятии последней &mdash;&nbsp;</span><strong><span class="">ничего не дёргается</span></strong><span class="">.</span></p>
</li>
</ul>
<p class="ds-markdown-paragraph"><strong><span class="">Изменения уже на веб-версии HubEx.</span></strong><span class="">&nbsp;Выбирайте заявки без лишних движений экрана &mdash; работа с массовыми действиями стала ещё комфортнее.</span></p>

<h3><strong><span class="">🔄 Выделение заявок сохраняется после массовых операций</span></strong></h3>
<p class="ds-markdown-paragraph"><span class="">Мы изменили логику работы списка заявок. Теперь после массовой смены стадии, редактирования или замены исполнителя все выбранные заявки&nbsp;</span><strong><span class="">остаются отмеченными</span></strong><span class="">. Больше не нужно заново искать и отмечать их для следующего действия.</span></p>
<h4><strong><span class="">В чём было неудобство?</span></strong></h4>
<p class="ds-markdown-paragraph"><span class="">Раньше после любой массовой операции список заявок полностью перезагружался, и все чекбоксы сбрасывались. Если нужно было выполнить несколько действий над одним и тем же набором заявок (например, сначала назначить исполнителя, потом сменить стадию), приходилось каждый раз заново их отмечать.</span></p>
<h4><strong><span class="">Что изменилось?</span></strong></h4>
<ul>
<li>
<p class="ds-markdown-paragraph"><span class="">После успешного выполнения массовой операции (</span><strong><span class="">смена стадии</span></strong><span class="">,&nbsp;</span><strong><span class="">смена исполнителя</span></strong><span class="">) все ранее выбранные заявки&nbsp;</span><strong><span class="">остаются отмеченными</span></strong><span class="">.</span></p>
</li>
<li>
<p class="ds-markdown-paragraph"><span class="">Список заявок обновляется (вы видите новые статусы, исполнителей), но выделение не сбрасывается.</span></p>
</li>
<li>
<p class="ds-markdown-paragraph"><span class="">Механика работает одинаково для&nbsp;</span><strong><span class="">табличного</span></strong><span class="">&nbsp;и&nbsp;</span><strong><span class="">карточного</span></strong><span class="">&nbsp;представлений.</span></p>
</li>
</ul>
<h4><strong><span class="">Важные нюансы</span></strong></h4>
<ul>
<li>
<p class="ds-markdown-paragraph"><span class="">Если после операции некоторые заявки перестали попадать под текущий фильтр (например, сменили стадию и исчезли из списка), они просто не отображаются, но выделение для остальных сохраняется.</span></p>
</li>
<li>
<p class="ds-markdown-paragraph"><span class="">При смене фильтра или переходе на другую страницу выделение сбрасывается (как и раньше).</span></p>
</li>
<li>
<p class="ds-markdown-paragraph"><span class="">При отмене операции (закрытие модального окна без подтверждения) выделение&nbsp;</span><strong><span class="">сохраняется</span></strong><span class="">.</span></p>
</li>
</ul>
<h4><strong><span class="">Преимущества для пользователей</span></strong></h4>
<ul>
<li>
<p class="ds-markdown-paragraph"><strong><span class="">Экономия времени</span></strong><span class="">&nbsp;&mdash; не нужно повторно выбирать заявки для цепочки действий.</span></p>
</li>
<li>
<p class="ds-markdown-paragraph"><strong><span class="">Удобство</span></strong><span class="">&nbsp;&mdash; можно последовательно менять стадию, исполнителя и другие параметры без лишних кликов.</span></p>
</li>
</ul>
<p class="ds-markdown-paragraph"><strong><span class="">Изменения уже на веб-версии HubEx.</span></strong><span class="">&nbsp;Попробуйте выполнить несколько массовых операций подряд &mdash; вы заметите разницу.</span></p>

<h3><strong><span class="">📋 Настройки колонок таблицы заявок теперь сохраняются&nbsp;для пользователей</span></strong></h3>
<p class="ds-markdown-paragraph"><span class="">Мы перенесли хранение настроек колонок табличного вида заявок из браузера в ваш профиль HubEx. Теперь порядок колонок и их видимость сохраняются надёжно и не сбрасываются при очистке кэша или смене устройства.</span></p>
<h4><strong><span class="">В чём было неудобство?</span></strong></h4>
<p class="ds-markdown-paragraph"><span class="">Раньше настройки колонок (какие поля видны, в каком порядке) хранились только в локальном хранилище браузера</span><span class="">.
