## Введение

**Заказчик** - это пользователь, который может создавать **Заявки** в мобильном приложении заказчика. **Заказчик** привязывается к компании-заказчику. С помощью настройки прав доступа для **Заказчика** ограничивается доступ к функционалу системы. Например, заказчик сможет видеть только свои **Объекты (оборудование)**, видеть **Заявки** только по своей компании и т.д.

Для Заказчика доступно мобильное приложение **HubEx заказчик**, которое можно скачать [для Android](https://play.google.com/store/apps/details?id=ru.hubex.customer) или [для iOS](https://apps.apple.com/ru/app/hubex-%D0%B4%D0%BB%D1%8F-%D0%B7%D0%B0%D0%BA%D0%B0%D0%B7%D1%87%D0%B8%D0%BA%D0%B0/id1386631658).

Прочтите подробную статью ниже или начните знакомство с темой с обучающего видеоролика **"Как создать нового пользователя в HubEx"**.

Создать нового заказчика можно в разделе левого бокового меню **"Пользователи" - "Заказчики"** по кнопке **"Создать заказчика"**.

![](https://wiki.hubex.ru/attachments/images/FAQ/USER/CreatingCustomerNew/CreatingCustomerNew1.png)

Карточка заказчика состоит из нескольких вкладок:

* **Общее**;
* **Квалификация**;
* **Прочее**.

![](https://wiki.hubex.ru/attachments/images/FAQ/USER/CreatingCustomerNew/CreatingCustomerNew2.png)

## Карточка заказчика. Вкладка "Общее"

При создании нового заказчика, на вкладке **"Общее"** обязательными для заполнения являются следующие поля:

* **Фамилия**;
* **Имя**;
* **Телефон** и (или) **Электронная почта**: это будущий логин для входа в аккаунт. После создания и сохранения карточки заказчика на указанный телефон/электронную почту придет сообщение/письмо с подтверждением регистрации и созданием пароля;
* **Компания**: к которой принадлежит заказчик;
* **Роль**: для заказчика необходимо выбрать одноименную роль **"Заказчик"**.

![](https://wiki.hubex.ru/attachments/images/FAQ/USER/CreatingCustomerNew/CreatingCustomerNew3.png)

Необязательные для заполнения поля:

* **Аватар (фото) заказчика**;
* **Отчество**;
* **Пол**;
* **Рабочий телефон**;
* **Другой телефон**;
* **Должность**.

Особенностью роли **"Заказчик"** является возможность создавать заявку, просматривать по ней основную информацию. **Заказчику** нельзя изменять стадии **Заявки**. Также у **Заказчика** не будет доступа ко всем **Заявкам** вашей компании.

![](https://wiki.hubex.ru/attachments/images/FAQ/USER/CreatingCustomer/CreateCustomer.jpg)

## Карточка заказчика. Вкладка "Квалификация"

На вкладке **"Квалификация"** необходимо указать **"Участок"**, по которому заказчик может подавать заявки.

![](https://wiki.hubex.ru/attachments/images/FAQ/USER/CreatingCustomerNew/CreatingCustomerNew5.png)

**"Участок"** - это классификация **"Объектов"**, **"Типов заявок"**, **"Сотрудников"**, **"Заказчиков"** по различным критериям: географическим, функциональным и т.д.

С помощью настройки **"Ролей"** по **"Участкам"** можно разграничить доступ **"Сотрудников"** и **"Заказчиков"** к работе с определенными **Объектами** и **Типами заявок**.

Например, можно создать участок «Электрооборудование» и отнести к нему все подходящие объекты. Далее создать тип заявки, например, «Ремонт электрооборудования», который будет иметь специфический жизненный цикл, и указать у него участок «Электрооборудование». Затем указать у заказчика тот же участок. Таким образом, этот заказчик сможет создавать заявки с типом «Ремонт электрооборудования» и выбирать объекты своей компании только по участку «Электрооборудования».

Для **"Cотрудника"**, **"Объекта"**, **"Заказчика"** и **"Типа заявки"** можно указать несколько **"Участков"**.

**"Участки"** создаются и настраиваются в консоли администратора. Подробнее читайте в статье: [Настройка участков](https://wiki.hubex.ru/docs/FAQ/RU/admin/Places.html.md), [Настройка ролей](https://wiki.hubex.ru/docs/FAQ/RU/admin/Roles.html.md#exrole).

## Карточка заказчика. Вкладка "Прочее"

На вкладке **"Прочее"** находится всего один реквизит: **"Бан пользователя"**. Он используется в случае запрета доступа в систему.

![](https://wiki.hubex.ru/attachments/images/FAQ/USER/CreatingCustomerNew/CreatingCustomerNew6.png)

После внесения всей необходимой информации не забудьте сохранить карточку нового заказчика.

![](https://wiki.hubex.ru/attachments/images/FAQ/USER/CreatingCustomerNew/CreatingCustomerNew7.png)

## Что представляет собой список заказчиков

Список всех заказчиков представляет собой карточный вид:

![](https://wiki.hubex.ru/attachments/images/FAQ/USER/CreatingCustomerNew/CreatingCustomerNew8.png)

Вы можете фильтровать список заказчиков, используя правое боковое меню:

![](https://wiki.hubex.ru/attachments/images/FAQ/USER/CreatingCustomerNew/CreatingCustomerNew9.png)

В списке заказчиков, карточка заказчика отображает:

* Аватар (фото);
* ФИО;
* Количество поданных заявок от заказчика;
* Кнопка вызова по телефону.

![](https://wiki.hubex.ru/attachments/images/FAQ/USER/CreatingCustomerNew/CreatingCustomerNew10.png)

Дополнительные настройки (три точки) дают возможность:

1. Копировать заказчика;
2. Отправить приглашение повторно. Если после ввода телефона/электронной почты при заполнении и сохранении новой карточки **Заказчика**, на указанный телефон/электронный адрес приглашение так и не пришло, то можно отправить его повторно. В карточке **Заказчика** или в общем списке всех **Заказчиков** нажмите на контекстное меню (три точки) и выберите пункт **Отправить приглашение повторно**.
3. Удалить заказчика.

![](https://wiki.hubex.ru/attachments/images/FAQ/USER/CreatingCustomerNew/CreatingCustomerNew11.png)

Для быстрого добавления **"Заказчиков"** в систему HubEx воспользуйтесь функцией  [импорта.](https://wiki.hubex.ru/docs/FAQ/RU/user/Import.html.md#workers)

Для выгрузки информации по **Заказчикам** из системы HubEx воспользуйтесь функцией  [экспорта.](https://wiki.hubex.ru/docs/FAQ/RU/user/Export.html.md#workers)

## Как перевести заказчика в сотрудники

Если вы случайно создали **"Заказчика"** вместо **"Сотрудника"**, то тип пользователя можно изменить.

Сделать это необходимо, так как права **"Заказчика"** и **"Сотрудника"** различаются. Например, "Заказчик" сможет переводить заявку только по стадиям ветки жизненного цикла **"Для заказчика"**, а по основной ветке жизненного цикла (для сотрудников) не сможет переводить заявку.

Для перевода **"Заказчика"** в **"Сотрудники"** нужно зайти в карточку **Заказчика**, нажать на **"Контекстное меню"** (3 точки), выбрать пункт **"Перевести к сотрудникам"**.

![](https://wiki.hubex.ru/attachments/images/FAQ/USER/CreatingCustomer/cust4.png)

В диалоговом окне нажмите кнопку **"Подтвердить"**.

![](https://wiki.hubex.ru/attachments/images/FAQ/USER/CreatingCustomer/cust5.png)

Обратите внимание, что при переводе **"Заказчика"** в **"Сотрудники"**, его роль не меняется. Роль нужно изменить вручную.
