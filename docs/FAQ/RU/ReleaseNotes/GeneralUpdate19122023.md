#### Важные доработки (триггеры увеломлений, редактирование объекта в МП и т.д.): обновление от 19.12.2023
<html>
<meta charset="utf-8">

</html>
<body>


<p>Команда HubEx представляет новые обновления, которые улучшат вашу работу в системе:</p>
<ol>
<li>редактирование поля <strong>Объект </strong>в <strong>Заявке</strong> в мобильном приложении;</li>
<li>упрощенный процесс входа в мобильное приложение для нового пользователя: завершение регистрации и создание пароля в мобильном приложении;</li>
<li>активация и деактивация триггеров уведомлений в web-приложении;</li>
<li>фильтрация <strong>Заявок</strong> по дате последнего изменения в web-приложении.</li>
</ol>
<h5>Редактирование Объекта в Заявке мобильном приложении</h5>
<p>Возможность изменить <strong>Объект</strong> на форме <strong>Заявки</strong> теперь доступна для мобильных приложений <strong>HubEx исполнитель</strong> и <strong>HubEx заказчик</strong>. Это позволит пользователям поддерживать актуальность данных в <strong>Заявках</strong> без обращения к диспетчеру web-приложения.</p>
<p>Если в <strong>Заявке</strong> для пользователя доступно редактирование поля <strong>Объект</strong>, то по нажатию на кнопку <strong>Х</strong>, можно очистить поле и выбрать другой <strong>Объект</strong>.</p>
<p>Обратите внимание! При очищении поля <strong>Объект</strong> также будет очищено поле <strong>Вид работ</strong>, так как <strong>Виды работ</strong> устанавливаются в карточках <strong>Объектов</strong>.</p>
<div><img style="margin: 0 auto; display: block; max-width: 100%;" src="https://wiki.hubex.ru/attachments/images/FAQ/RELEASENOTES/GeneralUpdate2023/Object.jpg" width="700" height="auto" /></div>
<p>При выборе нового <strong>Объекта</strong> также будет необходимо выбрать доступный для него <strong>Вид работ</strong>. Сохранение <strong>Заявки</strong> при этом произойдет автоматически.</p>
<div><img style="margin: 0 auto; display: block; max-width: 100%;" src="https://wiki.hubex.ru/attachments/images/FAQ/RELEASENOTES/GeneralUpdate2023/Object2.jpg" width="350" height="auto" /></div>

<h5>Завершение регистрации нового пользователя в мобильном приложении</h5>
<p>В HubEx существенно упрощен процесс установки пароля для инженеров, которые еще не подтвердили свою учетную запись. Теперь, когда инженер входит в систему в первый раз, он получит приглашение создать пароль прямо в мобильном приложении <strong>HubEx исполнитель</strong>. Этот удобный для пользователя подход обеспечивает беспрепятственный процесс завершения регистрации в системе.</p>
<div><img style="margin: 0 auto; display: block; max-width: 100%;" src="https://wiki.hubex.ru/attachments/images/FAQ/RELEASENOTES/GeneralUpdate2023/Password.jpg" width="700" height="auto" /></div>
<p>После ввода пароля необходимо подтвердить процесс прохождения регистрации через sms-код.</p>
<div><img style="margin: 0 auto; display: block; max-width: 100%;" src="https://wiki.hubex.ru/attachments/images/FAQ/RELEASENOTES/GeneralUpdate2023/Password2.jpg" width="700" height="auto" /></div>
<p>После завершения регистрации пользователь сможет входить в приложение как с помощью только что созданного пароля, так и по sms-коду. Подробнее читайте в статье: <a href="https://wiki.hubex.ru/docs/FAQ/RU/user/EnterToMob.html" target="_blank">Вход в мобильное приложение Заказчика и Сотрудника</a>.</p>

<h5>Активация и деактивация триггеров уведомлений</h5>
<p>Как вы уже знаете, триггеры уведомлений отвечают за настройку событий и уведомлений для пользователей. Теперь поддерживать актуальные триггеры можно с помощью активации и деактивации. Включить актуальный можно в меню консоли администратора <strong>Настройки оповещений - Триггеры уведомления&nbsp;</strong>с помощью переключателя (положение вправо - триггер активен, положение влево - триггер деактивирован). Подробнее читайте в статье: <a href="https://wiki.hubex.ru/docs/FAQ/RU/admin/Notifications.html" target="_blank">Настройка оповещений</a>.</p>
<div><img style="margin: 0 auto; display: block; max-width: 100%;" src="https://wiki.hubex.ru/attachments/images/FAQ/RELEASENOTES/GeneralUpdate2023/Trigger.jpg" width="800" height="auto" /></div>

<h5>Фильтрация Заявок по дате изменения</h5>
<p>В меню <strong>Заявки - Все заявки</strong> по кнопке фильтров теперь доступен новый фильтр <strong>По дате изменения</strong>.</p>
<p>Новый параметр фильтрации позволяет быстро идентифицировать и просматривать недавно обновленные <strong>Заявки</strong>. Таким образом можно всегда оставаться в курсе хода проекта.</p>
<div><img style="margin: 0 auto; display: block; max-width: 100%;" src="https://wiki.hubex.ru/attachments/images/FAQ/RELEASENOTES/GeneralUpdate2023/Filter.jpg" width="800" height="auto" /></div>
<p>
 HubEx продолжает развиваться и предоставлять пользователям интуитивно понятные и мощные функции. Используя текущие обновления, пользователи могут максимизировать производительность своего труда, оптимизировать рабочие процессы и с легкостью работать в системе.
</p>
