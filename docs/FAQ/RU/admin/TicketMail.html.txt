Интеграция с HubEx по электронной почте позволяет создавать **Заявки** с помощью получения электронных писем от ваших **Заказчиков**.

В системе HubEx предусмотрено два варианта подключения интеграции через электронную почту:

* **Подключение через пересылку**: данный способ предполагает настройку переадресации писем для
  вашего почтового ящика на специальный ящик,
  созданный индивидуально для вашего бизнес аккаунта. Заявки будут создаваться по единому шаблону для тех писем,
  для которых будет
  настроена пересылка.
* **Подключение почтового ящика через настройки imap**: данный способ предполагает более детальную
  настройку вашего
  почтового сервиса и подключение вашего ящика напрямую как адреса для сбора Заявок, создание индивидуальных
  шаблонов заявок под разных заказчиков и настройку двусторонней интеграции для отправки ответных писем из HubEx
  на электронную почту.

#### Настройка интеграции через пересылку

Настройка интеграции через пересылку состоит из нескольких этапов. Первым этапом является подготовка системы HubEx
для приема писем и преобразования их в **Заявки**.

##### Настройка HubEx для интеграции по пересылке

Подготовка HubEx к интеграции через пересылку состоит из двух шагов:

* **Шаг 1:** для интеграции по пересылке необходимо настроить шаблон заявки в пользовательском интерфейсе в меню **Заявки - Маркировка**. Шаблон для пересылки будет создан в вашем аккаунте по умолчанию, его можно изменить при необходимости. Заявка из письма будет создаваться в HubEx согласно этому шаблону. В рамках данной статьи будет краткая инструкция по редактированию шаблона. Подробно о шаблонах и QR-кодах читайте в статье: [Создание шаблона заявки, подача заявки по QR-коду](https://wiki.hubex.ru/docs/FAQ/RU/user/CreatingTaskTemplates.html.txt).

  Шаблон по умолчанию имеет название **Шаблон для получения заявок по электронной почте через
  пересылку.**

  ![](https://wiki.hubex.ru/attachments/images/FAQ/ADMIN/TicketMail/Integration23.jpg)

  На форме шаблона по умолчанию заполнены основные поля, некоторые из них можно отредактировать при
  необходимости:

+ поле **Описание QR-кода** - это название шаблона для интеграции
  с почтой;
+ по умолчанию шаблон создается неопубликованным и публиковать его для текущей интеграции необязательно.
  Подробнее о шаблоне и QR-кодах читайте в статье: [Создание шаблона
  заявки,
  подача заявки по QR-коду](https://wiki.hubex.ru/docs/FAQ/RU/user/CreatingTaskTemplates.html.txt);
+ флажок **Разрешить подачу заявки из Web** установлен по умолчанию, снимать его не нужно;
+ в поле **Тип заявки** по умолчанию заполнен тип, который был создан в системе первым. **Тип
  заявки** можно изменить в шаблоне. **Заявки** из писем будут создаваться с указанным в
  шаблоне типом;
+ поле **Заказчик** не заполнено по умолчанию, так как шаблон является общим для
  всех **Заказчиков**. Если к вам на почту будут поступать письма от одного конкретного
  **Заказчика**, то укажите его компанию, которая затем будет заполнена в **Заявке**;
+ поле **Объект|Оборудование** не заполнено по умолчанию. При необходимости укажите объект или
  оборудование, который будет указан в
  созданной **Заявке**, его можно будет изменить. Также для интеграции можно создать отдельный
  объект под
  названием **Для почты**, чтобы использовать его только в шаблоне для создания заявок с
  электронной почты. Таким образом легко можно будет отфильтровать **Заявки** в общем списке
  по
  исключительному объекту;
+ в поле **Вид работ** по умолчанию будет указан такой **Вид работ** в карточке
  которого установлен
  флажок **Основной вид работ**, он может быть только один в системе. **Вид работ** в
  созданной **Заявке** можно будет изменить;
+ **Критичность** в шаблоне будет установлена та, в карточке которой установлен флажок **По
  умолчанию**, это поле
  также можно будет менять в созданной по письму **Заявке**;
+ поле **Описание заявки** по умолчанию остается пустым, текст из письма при подаче через
  интеграцию заполняется как описание;
+ **Приглашение** для интеграции с почтой указывать не нужно. Приглашения используются при
  подаче
  **Заявок** по QR-кодам. Подробнее читайте в статье: [Подача заявок по QR-коду без
  приложения, самостоятельная регистрация нового пользователя](https://wiki.hubex.ru/docs/FAQ/RU/user/SelfRegister.html.txt).
+ Если вы вносили изменения в поля шаблона, то нажмите кнопку **Сохранить**.

![](https://wiki.hubex.ru/attachments/images/FAQ/ADMIN/TicketMail/Integration25.jpg)

  и

  ![](https://wiki.hubex.ru/attachments/images/FAQ/ADMIN/TicketMail/EmailSettings3.jpg)

  Придумайте название для пароля (например, Пароль для почты). Затем нажмите кнопку **Далее**.

  ![](https://wiki.hubex.ru/attachments/images/FAQ/ADMIN/TicketMail/EmailSettings4.jpg)

  В новом окне будет сгенерирован специальный пароль. Его необходимо скопировать по нажатию на иконку
  копирования (два квадратика). При копировании вы увидите уведомление, что идет процесс копирования.
  Скопированный пароль необходимо сохранить в заметках или текстовом документе (формата .txt, .docx) на
  вашем
  компьютере (или мобильном телефоне). Этот пароль будет необходим в дальнейшем для настроек интеграции в
  HubEx.

  ![](https://wiki.hubex.ru/attachments/images/FAQ/ADMIN/TicketMail/EmailSettings5.jpg)

  ![](https://wiki.hubex.ru/attachments/images/FAQ/ADMIN/TicketMail/EmailSettings6.jpg)

  Если вы создали пароль для приложений и не скопировали (не сохранили) его, то необходимо создать новый
  пароль
  по схеме, описанной выше.

##### Настройка HubEx для интеграции

Вторым этапом настройки интеграции является настройка HubEx:

* **Шаг 1:** для интеграции необходимо создать шаблон заявки в пользовательском
  интерфейсе в
  меню **Заявки -
  Маркировка** по кнопке
  **Создать QR-код**. Заявка из письма будет создаваться в HubEx согласно этому шаблону. В рамках
  данной статьи будет краткая инструкция по созданию и заполнению шаблона. Подробно о шаблонах и QR-кодах
  читайте
  в статье: [Создание шаблона заявки,
  подача заявки по QR-коду](https://wiki.hubex.ru/docs/FAQ/RU/user/CreatingTaskTemplates.html.txt).

На форме шаблона заполните основные поля:

+ поле **Описание QR-кода** - это название шаблона, так как мы создаем шаблон для интеграции
  с почтой, то в названии можно написать, например, Шаблон для почты;
+ установите флажок **Опубликовать**: флажок отвечает за доступность шаблона и QR-кода (для
  настроек интеграции QR-код не нужен). Подробнее о шаблоне и QR-кодах читайте в статье: [Создание шаблона
  заявки,
  подача заявки по QR-коду](https://wiki.hubex.ru/docs/FAQ/RU/user/CreatingTaskTemplates.html.txt). Если этот флажок установлен
  и
  указано **Приглашение** (поле внизу формы), то незарегистрированные ранее пользователи
  смогут подать
  заявку по
  данному
  шаблону и пройти регистрацию в HubEx;
+ установите флажок **Разрешить подачу заявки из Web**;
+ укажите самый востребованный **Тип заявки** в созданной по шаблону **Заявке**
  его можно будет изменить;
+ в поле **Заказчик** укажите компанию-заказчика, от лица которой будут
  приходить электронные
  письма. Для разных заказчиков необходимо создавать индивидуальные шаблоны, если это необходимо.

  Обратите внимание! Вы можете сделать общий шаблон для всех заказчиков, без привязки к конкретному.
  Для этого оставьте поле **Заказчик** и **Объект** пустыми;
+ в поле **Объект|Оборудование** укажите объект или оборудование, который будет указан в
  созданной **Заявке**, его можно будет изменить. Также можно создать отдельный объект под
  названием **Для почты**, чтобы использовать его только в шаблоне для создания заявок с
  электронной почты. Таким образом легко можно будет отфильтровать **Заявки** в общем списке
  по
  исключительному объекту;

  Обратите внимание! Вы можете сделать общий шаблон для всех заказчиков, без привязки к конкретному.
  Для
  этого в поле **Заказчик** необходимо выбрать вашу компанию (от лица, которой вы
  работаете),
  а в поле
  **Объект** выбрать объект **Для почты** (Объект **Для почты**
  должен быть создан до создания шаблона). Таким образом, например, диспетчер сможет обработать все
  входящие Заявки с фильтром по вашей компании и объекту **Для почты**.
+ в поле **Вид работ** укажите самый популярный вариант, если их на объекте
  несколько. Выбор **Вида работ**
  осуществляется из тех, что установлены в карточке **Объекта**. Вид работ при подаче заявки
  можно будет
  изменить;

Обратите внимание! У шаблона обязательно должен быть выбран **Объект** и **Вид
работ**, иначе его нельзя будет выбрать в настройках интеграции.

+ **Критичность** заполняется на усмотрение, эти поля
  также можно будет менять при
  подаче **Заявки**;
+ поле **Описание заявки** оставьте пустым;
+ **Приглашение** для интеграции с почтой указывать не нужно. Приглашения используются при
  подаче
  **Заявок** по QR-кодам. Подробнее читайте в статье: [Подача заявок по QR-коду без
  приложения, самостоятельная регистрация нового пользователя](https://wiki.hubex.ru/docs/FAQ/RU/user/SelfRegister.html.txt).

![](https://wiki.hubex.ru/attachments/images/FAQ/ADMIN/TicketMail/Template.jpg)

После заполнения полей шаблона нажмите кнопку **Сохранить**. После сохранения будет
сгенерирован
QR-код. QR-код не используется для интеграции. А сам шаблон пригодится позже для настроек в
консоли администрирования.

* **Шаг 2:** активация сервисной учетной записи.

Активировать сервисную учетную запись можно в консоли администрирования в меню **Интеграция - Служебные
пользователи**. Далее будут представлены настройки подходящие для интеграции с почтой.
Дополнительно про сервисные учетные записи читайте в статье: [Служебные пользователи](https://wiki.hubex.ru/docs/FAQ/RU/admin/ServiceUsers.html.txt).

Переключите тумблер **Пользователя API**
вправо и установите необходимые настройки:

+ выберите **Роль**, которая наделена максимальными полномочиями для работы с
  **Заявками**
  и
  **Объектами**. При интеграции с выбранных для **Пользователя API**
  роли будут считываться полномочия. Обычно полными правами наделены Администраторы или Системные
  администраторы (можно указать обе эти **Роли**, если они есть в системе);
+ также выберите **Участки**, для которых будет доступна интеграция по почте. Рекомендуем
  выбрать все участки. Подробнее об
  участках читайте в статье: [Настройка участков](https://wiki.hubex.ru/docs/FAQ/RU/admin/Places.html.txt);
+ поле **Токен доступа** оставьте незаполненным, для интеграции с электронной почтой токен не
  нужен.
  Токен (или ключ) нужен при настройке интеграции с Битрикс24 с помощью Albato. Подробнее читайте в
  статье: Интеграция с Битрикс24 с помощью Albato.

![](https://wiki.hubex.ru/attachments/images/FAQ/ADMIN/TicketMail/ServiceUsers.jpg)

* **Шаг 3:** перейдите в разделе **Интеграция - Интеграция по электронной
  почте** и нажмите кнопку **Подключить эл. почту**.

Заполните следующие поля:

+ **Адрес сбора заявок**: укажите тот электронный адрес, на который будут поступать письма с
  заявками (это та почта, для которой мы делали настройки в [первом разделе статьи](https://wiki.hubex.ru/docs/FAQ/RU/admin/TicketMail.html.txt#set1)). Мы
  рекомендуем использовать для интеграции почтовые ящики yandex.ru, так как другие сервисы могут
  работать
  нестабильно. Если у вас другой почтовый сервис, то предлагаем вам создать почтовый ящик на yandex.ru и
  установить на него пересылку с вашего почтового сервиса;
+ в полях **Логин** и **Пароль/Ключ** укажите логин (почтовый ящик,
  на
  который будут поступать письма) и [пароль приложений](https://wiki.hubex.ru/docs/FAQ/RU/admin/TicketMail.html.txt#password), созданный ранее в
  настройках
  аккаунта Яндекс;
+ поля **Хост**, **Порт**, **Сетевой протокол**, а также **Протокол
  безопасности** заполните в соответствии с настройками почтового сервера;

для Yandex эти настройки будут: **Хост**= imap.yandex.ru, **Порт**= 993,
**Протокол
безопасности** = SSL/TLS;

+ далее нажмите кнопку **Добавить адрес** и введите адрес электронной почты, с которого будут
  отправлять
  **Заявки**. Вы
  можете добавить несколько адресов. Для каждого заказчика свой адрес электронной почты и шаблон заявки;
+ **Отправитель** - здесь может быть указан конкретный e-mail адрес, у которого будет право
  создания
  **Заявки**.
  Либо вы можете ввести значение
  "\*@yandex.ru" (без кавычек), тогда у всех почтовых ящиков \*@yandex.ru будет доступ к подаче
  **Заявок**
  через e-mail. Либо ввести "\*@\*" (без кавычек) для получения писем с любых адресов любых доменов.

  Обратите внимание! Если вы создаете общий шаблон заявки, без привязки к [конкретному
  заказчику](https://wiki.hubex.ru/docs/FAQ/RU/admin/TicketMail.html.txt#customer), то указывайте в поле **Отправитель** вариант "\*@\*" (без кавычек) для
  получения писем от любых электронных адресов;
+ в поле **Шаблон заявки** выберите из списка шаблон, созданный в [первом
  шаге](https://wiki.hubex.ru/docs/FAQ/RU/admin/TicketMail.html.txt#template);
+ для получения писем с указанного адреса установите переключатель **Разрешить получение
  писем**
  вправо;
+ дополнительно можно подключить [двустороннюю интеграцию](https://wiki.hubex.ru/docs/FAQ/RU/admin/TicketMail.html.txt#twowayintegration) и отправлять
  сообщения **Заказчику**
  на
  почту из чатов по **Заявкам**, для этого переведите переключатель **Возможность
  ответа
  заказчику через чат** вправо;
+ **Настройка регулярных выражений**: данный блок настроек необходим для создания правил
  заполнения
  **Заявки** по тексту
  электронного письма (настройка парсинга). Например, из электронного письма при создании
  **Заявки** в системе
  можно
  будет
  заполнить **Объект**, **Вид работ**, **Срок
  закрытия** и т.д. Настройка регулярных выражений является дополнительной опцией. Для создания
  регулярных выражений обратитесь в службу поддержки по адресу [help@hubex.ru](mailto:help@hubex.ru)

  Например, настройка регулярных выражений может выглядеть следующим образом (на скриншоте представлена
  частичная настройка).

  ![](https://wiki.hubex.ru/attachments/images/FAQ/ADMIN/TicketMail/RegularPhrase2.jpg)

  При использовании регулярных выражений электронное письмо, которое отправляют заказчики, будет
  выглядеть следующим образом.

  ![](https://wiki.hubex.ru/attachments/images/FAQ/ADMIN/TicketMail/Mail3.jpg)

На этом настройка подключения почты закончена. После заполнения всех данных нажмите кнопку
**Сохранить**.

Обратите внимание на тумблер **Включения/Выключения** интеграции с электронной почтой. В
положении
**Выключено** вы больше не
сможете подавать заявку через почту, однако данная интеграция не будет удалена из системы. При
необходимости
вы
сможете снова ее включить.

![](https://wiki.hubex.ru/attachments/images/FAQ/ADMIN/TicketMail/Email3.jpg)

Как работает двусторонняя интеграция?

1. Для работы двусторонней интеграции выполните настройки интеграции по данной статье, описанные
   выше;
2. далее Заказчик должен отправить письмо на адрес сбора заявок с описанием **Заявки**.

Обратите внимание! Если вы прочитаете полученное письмо на адресе сбора заявок (вашей почте), то оно
не будет создано в HubEx как **Заявка**;

![](https://wiki.hubex.ru/attachments/images/FAQ/ADMIN/TicketMail/Mail.jpg)

3. в HubEx создается **Заявка**;

![](https://wiki.hubex.ru/attachments/images/FAQ/ADMIN/TicketMail/Ticket.jpg)

4. в **Заявке** во вкладке **Сообщения** в чате с заказчиком можно вести
   переписку (задавать уточняющие вопросы, просить прислать фото, видео и т.д.) и каждое сообщение
   для
   **Заказчика** будет отправлено не только в чат, но и сформировано как ответное
   письмо
   на адрес, с которого была отправлена **Заявка**.

![](https://wiki.hubex.ru/attachments/images/FAQ/ADMIN/TicketMail/Chat.jpg)

![](https://wiki.hubex.ru/attachments/images/FAQ/ADMIN/TicketMail/Mail2.jpg)

##### Подача заявки через электронную почту

1. Напишите письмо на адрес электронной почты, который был указан как **Адрес сбора заявок**.
2. Введите тему сообщения и текст письма.

   Обратите внимание! Если у вас заданы **Настройки регулярных
   выражений** (см.выше), то тему и текст письма нужно будет вводить согласно заданных настроек.
   Настроить регулярные выражения и дать рекоменадции по их использованию вам также помогут специалисты
   нашей службы поддержки по адресу [help@hubex.ru](mailto:help@hubex.ru).

   Без настройки регулярных выражений текст письма можно заполнять в свободной форме (как в примере ниже или
   в примере с [двусторонней интеграцией](https://wiki.hubex.ru/docs/FAQ/RU/admin/TicketMail.html.txt#twowayintegration)).

![](https://wiki.hubex.ru/attachments/images/FAQ/ADMIN/TicketMail/emailtick5.jpg)

После отправки письма, в системе будет создана **Заявка**. На почту вы получите уведомление о ее
создании. Вы можете написать ответ на письмо, в таком случае он будет указан в системе, как сообщение по
**Заявке**.

![](https://wiki.hubex.ru/attachments/images/FAQ/ADMIN/TicketMail/emailtick6.png)

Обратите внимание! Если **Заявка** была подана с электронного адреса, который не был зарегистрирован
в
системе, то
пользователю, создавшему **Заявку**, придет письмо со ссылкой для регистрации в системе.
