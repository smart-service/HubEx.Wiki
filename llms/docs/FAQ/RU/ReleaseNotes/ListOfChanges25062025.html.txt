## 🚀 Добавление вкладки "Договоры" в карточку объекта

## 📌 Описание изменений

Мы рады сообщить о новой функциональности в карточке объекта! Теперь доступна вкладка **"Договоры"**, которая позволяет:

* 📋 Просматривать список договоров, связанных с объектом
* 🔍 Открывать детальную информацию о договоре в боковом окне
* ➕ Создавать новые договоры с автоматическим заполнением данных компании и объекта

**🔧 Как это работает**

**📂 Вкладка "Договоры"**

* В списке вкладок карточки объекта появилась новая вкладка **"Договоры"**
* При переходе на вкладку система автоматически загружает список договоров, связанных с текущим объектом

**📊 Таблица договоров**

* Отображается в том же формате, что и во вкладке "Компании > Договоры"
* Содержит основную информацию:
  + Название и номер договора
  + Описание
  + Условия договора
  + Срок действия

**🖱️ Взаимодействие**

* **Клик по строке** - открывает боковое окно с детальной информацией о договоре
* **Кнопка "Создать договор"** - открывает форму создания нового договора с автоматически заполненными полями:
  + Компания (владелец объекта)
  + Объект (текущее оборудование)

**🎯 Преимущества изменений**

* ⏱️ Быстрый доступ ко всем договорам объекта без перехода в другие разделы
* 📑 Удобное управление договорами прямо из карточки оборудования
* 🛠️ Экономия времени при создании новых договоров благодаря автозаполнению

## 🚀 Улучшенный просмотр истории обслуживания с иерархией заявок

## 📌 Описание изменений

Мы значительно улучшили функциональность вкладки **"История обслуживания"** в карточке объекта! Теперь вы можете:

* 🌳 Видеть полную иерархию заявок "Родитель-Дочерняя" по выбранному объекту
* 🏷️ Легко идентифицировать связи между заявками

## 🌟 Ключевые возможности

### 🔗 Иерархия заявок

* Родительские заявки отображаются с возможностью раскрытия списка дочерних
* Любая заявка с заполненным полем **"Родительская заявка"** автоматически считается дочерней
* Поддерживается **неограниченный уровень вложенности** (дочерние заявки могут иметь свои дочерние заявки)

### 🔍 Параметры иерархии

* При **отключенной** опции "С дочерними объектами" отображаются только заявки текущего объекта
* При **включенной** опции "С дочерними объектами" список дополняется заявками из дочерних объектов
* Система **не показывает** недоступные пользователю заявки вместе со всей их нижестоящей иерархией

### 👁️ Визуализация

* Родительские заявки помечены специальными иконками:
  + 🔽 Свернутый список дочерних заявок
  + 🔼 Раскрытый список дочерних заявок
* Заявки без дочерних элементов отображаются без иконки управления

## ⚠️ Важно

Если родительская заявка недоступна пользователю, **вся ее иерархия** (включая дочерние заявки) автоматически скрывается из просмотра.

## 🚀 Массовое изменение стадий заявок

## 📌 Описание изменений

Мы рады представить новую функциональность — **массовое изменение стадий заявок**! Теперь вы можете:

* 🔄 Изменять стадии у **нескольких заявок одновременно** (до 500 за один раз)
* 🎯 Выбирать стадию сразу для группы заявок по типу заявки и текущей стадии
* 📊 Удобно управлять переводами через интуитивно понятный интерфейс

## 🌟 Ключевые возможности

### 1️⃣ Выбор заявок

* Выбирайте заявки **вручную** на одной или нескольких страницах
* Используйте **фильтры** для быстрого отбора нужных заявок
* В панели действий появилась новая кнопка **«Изменить стадию»**

### 2️⃣ Управление переводами

* Заявки автоматически группируются **по типам и текущим стадиям**
* Раскрывайте группы для просмотра деталей:
  + Номер заявки
  + Описание
  + Дата создания
* Для каждой группы можно выбрать **новую стадию** из доступных переходов

### 3️⃣ Гибкие сценарии работы

* Если для группы недоступен переход — система покажет сообщение:
  *«Изменение стадии этих заявок недоступно»*
* Можно переводить **не все заявки** — оставшиеся сохранят текущую стадию
* **Отмена** не сбрасывает выбранные заявки
* Пользователю отображаются **только доступные** ему стадии

### 4️⃣ Уведомления и история

* После сохранения система работает **в фоновом режиме**
* Вы получите уведомление о результате:
  *«Массовый перевод заявок по стадиям завершён»*
* Во вкладке **«История изменений»** каждой заявки будет отражен перевод
