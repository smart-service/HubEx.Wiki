<p>В HubEx представлены аналитические отчеты реализованные на платформе Power BI. С помощью бизнес-аналитики вы
    можете анализировать данные, получать ценные сведения для принятия бизнес-решений.</p>
<p>Каждый аналитический отчет реализован в виде панели с различными графиками, диаграммами, сводными
    таблицами и показателями.</p>
<p><Strong>Отчет по сотрудникам</Strong> в HubEx дает возможность проанализировать данные, собранные по
    <Strong>Сотрудникам</Strong> и <Strong>Заявкам</Strong>. С помощью этих визуализаций
    вы
    сможете
    ответить на
    вопросы - </p>

<p><Strong>Отчет по сотрудникам</Strong> находится в меню <Strong>Аналитика - Отчет по сотрудникам</Strong>. Данные
    в отчете обновляются один раз в
    сутки.
    Дата
    обновления отображается слева от наименования отчета. </p>
<div>
    <img style="margin: 0 auto; display: block; max-width: 95%;"
         src="/attachments/images/FAQ/USER/EngineersAnalitics/XXXXXXXXXXXXXXXXXXXXXXXX.jpg"/>
</div>
<p>Представление панели отчетов основывается на данных в <Strong>Заявках</Strong> в разрезе <Strong>Сотрудников</Strong>.
    Сотрудники – пользователи, которые могут создавать и выполнять Заявки. Сотрудником может являться диспетчер вашей
    организации, который создает заявки в web-приложении, руководитель вашей
    организации, сервисный специалист (исполнитель), который работает по Заявкам. В контексте данного отчета будут
    рассматриваться данные именно по специалистам, которые хотя бы раз были назначены на Заявку.

    Подробнее читайте в статьях: <a
            href="https://wiki.hubex.ru/docs/FAQ/RU/user/CreatingObjects.html#parentchild">Создание пользователя вашей
        системы
        объектов</a>, <a
            href="https://wiki.hubex.ru/docs/FAQ/RU/user/CreatingTicket.html">Создание заявки</a>.
</p>
<p>В левом верхнем углу вы можете указать период отображения данных в отчете.</p>
<p>С помощью кнопки <Strong>Фильтров</Strong> можно выбрать <Strong>Тип заявки</Strong>,
    <Strong>Участок</Strong> и т.д. Данный фильтр
    будет применен ко всем диаграммам и показателям в панели отчетов.</p>

<p>Отчет по сотрудникам представляет из себя три вкладки, разбитых по тематикам и содержащих в себе различные графики,

<ol>

<h5 id="first">Группа отчетов и показателей по Заявкам и SLA</h5>
<p>В верхней части представлены основные показатели по количеству Заявок за выбранный период по всем исполнителям:</p>
<ol>
    <li>Количество созданных заявок;</li>
    <li>Количество закрытых заявок;</li>
    <li>Количество открытых заявок;</li>
    <li>Количество переназначенных заявок - количество Заявок, с которых исполнители снимали свое
        назначение/отказывались от Заявки;
    </li>
    <li>% закрытых с просрочкой заявок: просроченной считается Заявка, которую закрыли позже установленного в ней
        крайнего срока закрытия. Подробнее читайте в статье: Создание заявки;
    </li>
    <li>Рейтинг - средний рейтинг по всем специалистам. Подробнее читайте в статье: <a
            href="https://wiki.hubex.ru/docs/FAQ/RU/user/Rating.html">Оценка заявок и рейтинг сотрудников</a>.
    </li>
</ol>
<p>Детальная информация по показателям в разрезе исполнителей представлена ниже в <a href="#table">таблице <strong>Показатели
    по
    сотрудникам</strong></a>.</p>

<p>Далее представлены две визуализации:</p>
<ul>
    <li><strong>Количество закрытых заявок и количество нарушений SLA</strong>. Данная диаграма представляет рейтинг
        исполнителей по
        закрытым Заявкам: синие столбцы - количество закрытых Заявок исполнителем за выбранный период, красные
        столбцы - % из этих Заявок, закрытых с просрочкой.

        <p>Данный отчет можно <a href="fourth">расшифровать по каждому сотруднику в отдельности</a>. Щелкните на исполнителя и нажмите значок
            стрелки справа на панели настроек.</p>
        <div>
            <img style="margin: 0 auto; display: block; max-width: 95%;"
                 src="/attachments/images/FAQ/USER/ObjectsAnalitics/XXXXXXXXXXXXXXX.jpg"/>
        </div>
        <p>Перед вами откроется детальный отчет по сотруднику. С помощью расшифровки по аналитическим разрезам можно
            ответить на впоросы: В чем причина закрытия заявок с просрочкой? Например, можно выбрать разрезы - Закрытые
            заявки - Закрытые с просрочкой - Клиенты - Объекты и проанализировать полученные Заявки.</p>
    </li>
    <li><strong>Зависимость нарушений SLA от среднего количества открытых заявок</strong>. Данная визуализация
        представляет собой зависимость закрытия Заявок с просрочкой от загрузки специалиста. С ее помощью вы сможете
        ответить на вопросы: Кто из специалистов может быть перегружен работой? Кому из исполнителей необходима помощь в
        работе с системой?
        <p>На данной визуализации представлено две оси (два показателя): <strong>Среднее кол-во открытых заявок</strong>
            - это среднее
            количество Заявок в день, которые назначены на специалиста, т.е. его загрузка; <strong>% закрытых заявок с
                просрочкой</strong> - отражает % Заявок, которые были закрыты с просрочкой. Каждый исполнитель на
            графике отображается
            точкой. Диаметр этой точки зависит от количества закрытых Заявок за выбранный период - чем крупнее точка,
            тем больше исполнитель закрыл Заявок.
        </p>
        <p>По каждой оси графика высчитывается средний показатель по всем исполнителям. Эти средние показатели образуют
            4
            квадранта - категории, на которые делятся исполнители: </p>
        <ol>
            <li>Первая категория (верхний правый квадрант): сюда попадают исполнители, у которых оба показателя
                завышены. Такие исполнители перегружены работой, не успевают закрывать Заявки. Заявки с таких
                исполнителей можно переназначить на исполнителей Третьей категории;
            </li>
            <li>Вторая категория (верхний левый квадрант): сюда попадают исполнители, у которых количество Заявок меньше
                среднего, но они стабильно закрывают их с нарушениями SLA (закрытие с просрочкой). Нужно искать причины,
                почему исполнители нарушают сроки закрытия при загруженности меньше остальных (возможно, нужна
                дополнительная помощь или обучение по работе HubEx с Заявками, возможно эти исполнители выполняют Заявки
                не по своему Виду работ, в связи с чем тратят на них большое количество времени и т.д.);
            </li>
            <li>Третья категория (нижний левый квадрант): сюда попадают исполнители с минимальными нарушениями SLA при
                загруженности ниже среднего. Возможно, на таких исполнителей следует переназначать Заявки с исполнителей
                Первой категории. Будьте внимательны, здесь могут оказаться специалисты, которые работают у вас
                на непостоянной основе, приходящие мастера и т.д. В таком случае, распределить на них Заявки
                специалистов из Первой категории скорее всего не получится (у приходящих исполнителей могут быть
                индивидуальные графики работ, и мало ЗАявок в принципе);
            </li>
            <li>Четвертая категория (нижний правый квадрант): сюда попадают исполнители с большим количеством Заявок и
                успевающие закрывать их в срок. Специалисты из данной категории нуждаются в корректировке меньше всех
                прочих.
            </li>
        </ol>
    </li>
</ul>

<p>Таблица <strong> Среднее количество выполненных заявок по дням недели</strong> показывает среднее количество Заявок
    закрываемых специалистом на каждый день недели. Ячейки с минимальная загруженностью окрашиваются в светло-голубой
    цвет, с максимальной загруженностью - в темно-синий. С помощью данной таблицы вы сможете определить, в какие дни
    ваша сервисная служба загружена больше всего. </p>
<p>Если вы указали период один месяц, значит данные в таблице будут браться по всем дням недели в течение этого месяца
    (все понедельники, все вторники и тд). Равномерное распределение закрытых Заявок означает, что планирование и
    назначение Заявок
    считается корректным.</p>

<h5 id="second">Группа отчетов и показателей по отработанному времени</h5>
<p>В верхней части представлены основные показатели по отработанному времени за выбранный период по всем
    исполнителям:</p>
<ol>
    <li>Отработано всего (в часах) - сумма рабочего времени исполнителей по графикам работ. График работ устанавливается
        в карточке Сотрудника. Подробнее читайте в статье: <a
                href="https://wiki.hubex.ru/docs/FAQ/RU/user/Schedule.html">График работы исполнителей</a>;
    </li>
    <li>Выполнено работ (в часах) - это сумма отработанного времени, которое указывается на вкладке Выполнение в Заявке.
        Подробнее читайте в статье: <a href="https://wiki.hubex.ru/docs/FAQ/RU/user/AttachingFiles.html">Прикрпеление
            выполненных работ</a>;
    </li>
    <li>Переработки (в часах) - считаются как часы, указанные во вкладке Выполнение в Заявке, но не входящие в рамки
        Графика работы сотрудников;
    </li>
    <li>Средняя переработка в день (в часах) - предыдущий показатель делится на количество исполнителей;</li>
    <li>Средняя загрузка в день (в часах) - сколько в среднем отмечается часов во вкладке Выполнение в Заявке.</li>
</ol>
<p>Детальная информация по показателям в разрезе исполнителей представлена ниже в <a href="#table2">таблице <strong>Временные
    показатели
    по
    сотрудникам</strong></a>.</p>

<p>Далее представлены две визуализации:</p>
<ul>
    <li>Гистограмма Выполненные работы и переработки отображает количество отработанных часов (синие столбцы) в разрезе
        исполнителей.
        Таким образом, вы можете понять кто из специалистов является ценным сотрудником и приносит больше денег
        компании. Красными столбцами отображаются переработанные часы. Например, график работы исполнителя с 9.00 до
        18.00, в Заявке во вкладке Выполнение он отмечает выполнение работы с 18.00 до 19.00 - этот час, вне его
        графика, будет считаться переработкой, даже если до этого времени у него не было внесено никаких больше
        выполненных работ.
        <p>Для корректного отображения данных необходимо устанавливать графики работ в карточка Сотрудников. Подробнее
            читайте в статье: <a
                    href="https://wiki.hubex.ru/docs/FAQ/RU/user/Schedule.html">График работы исполнителей</a>.</p>
    </li>
    <li>На гистограмме Динамика отработанных часов и переработок отображаются отработанные часы и переботки исполнителей
        в сравнении с графиком работ.
        <p>Данный график полезно анализировать в разрезе конкретного исполнителя. Выбрать его
            можно щелкнув по исполнителю в гистограмме Выполненные работы и переработки. Таким образом, на гистограмме
            появится линия с графиком работ выбранного исполнителя.</p>
    </li>

</ul>


<p>Таблица Среднее количество переработок по дням недели показывает среднее количество переработанного времени

    цвет, с максимальным - в красный. С помощью данной таблицы вы сможете определить, в какие дни

<p>Если вы указали период один месяц, значит данные в таблице будут браться по всем дням недели в течение этого месяца

<h5 id="third">Показатели и KPI</h5>
<p>В верхней части расположены основные показатели:</p>
<ul>
    <li>количество Мобильных сотрудников - количество выездных специалистов. Флажок Мобильный сотрудник устанавливается
        в карточке Пользователя. Подробнее читайте в статье: Создание пользователя вашей системы;
    </li>
    <li>количество Офисных сотрудников - количество специалистов, у которых не стоит в карточке Сотрудника флажок
        Мобильный сотрудник;
    </li>
    <li>количество Открытых заявок - количество открытых Заявок на всех исполнителях;</li>
    <li>Коэффициент полезной загрузки исполнителей - коэффициент, возволяющий оценить загрузку исполнителей. Подробнее
        представлен в визуализации слева.
    </li>
    <li>количество Заявок на человека в день - среднее количество поступающих Заявок на одного исполнителя;</li>
    <li>Рейтинг - средний рейтинг по исполнителям. Рейтинг устанавливается в рамках каждой Заявки. Подробнее читайте в
        статье: <a
                href="https://wiki.hubex.ru/docs/FAQ/RU/user/Rating.html">Оценка заявок и рейтинг сотрудников</a>.
    </li>
</ul>


<p>Слева представлена визуализация по показателю <Strong>Коэффициент полезной загрузки исполнителей</Strong>
    (Utilization

<p><Strong>HubEx рейтинг</Strong> рассчитывается индивидуально по каждому исполнителю на основе четырех показателей:</p>
<ul>

    <li>% закрытых заявок с нарушением SLA;</li>

    <li>Рейтинг (оценка сотрудников заказчиками).</li>
</ul>

    выполнении Заявок. </p>

<h5 id="fourth">Отчет-карточка сотрудника</h5>
<p>В отчет-карточку по сотруднику можно перейти из любой вкладки Отчета по сотруднику, щелкнув на его ФИО в любой визуализации и нажав на кнопку значок
    стрелки справа на панели настроек.</p>
<p>Подробный отчет по сотруднику содержит в себе ключевые показатели по Заявкам и отработанному времени:</p>
<uL>
    <li>количество Открытых заявок;</li>
    <li>количество Закрытых заявок;</li>
    <li>процент Закрытых с просрочкой</li>
    <li>Заявок в день;</li>
    <li>Выполнено работ (в часах);</li>
    <li>Переработки (в часах);</li>
    <li>Рейтинг.</li>

</uL>

<p>Также по сотруднику представлен график динамики изменения HubEx рейтинга за выбранный период времени.</p>

<p>История перемещений представляет (геотрекинг) отображает перемещения каждого сотрудника, у которого включена
    функция использования геопозиции. Включить функцию отслеживания исполнителя можно в карточке Сотрудника (флажок Отслеживать местоположение сотрудника).
    С помощью геотрекинга вы можете оценить эффективность работы мобильных
    исполнителей: время в пути, на заявке, остановки, простои. </p>
