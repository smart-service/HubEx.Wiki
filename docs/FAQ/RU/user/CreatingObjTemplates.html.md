**Компания-владелец**, **Тип объекта**, **Участок**, **Вид
работ**, **Ответственный за объект**. С помощью шаблона объектов можно быстро
создавать новые объекты (оборудование) с помощью
мобильного приложения исполнителя в меню **Приемка и маркировка**.

Обратите внимание! При создании объектов (оборудования) в мобильном приложении исполнителя новые
**Объекты** будут
созданы неопубликованными (по ним нельзя будет создавать заявки). Как опубликовать **Объект**
написано [ниже](https://wiki.hubex.ru/docs/FAQ/RU/user/CreatingObjTemplates.html.md#publication).

##### Создание шаблона объектов (оборудования) в web-приложении

Создать шаблон объектов (оборудования) можно в web-приложении в меню **Объекты|Оборудование - Шаблон
объектов** и
оборудования.

Шаблон объектов необходим для ввода новых объектов или оборудования с помощью мобильного приложения исполнителя.

Например, вы планируете обслуживать заказчика, у которого большой список оборудования. Чтобы внести все оборудование
можно создать общий шаблон и отправить мобильного сотрудника (сервисного специалиста) для создания этого
оборудования по шаблону.

Рассмотрим заполнение полей вкладки **Объект|Оборудование**:

* **Название шаблона** будет отображаться в мобильном приложении исполнителя при создании объектов по
  шаблону в меню
  **Приемка и маркировка**;
* **Описание шаблона** заполняется в свободной форме, не обязательно для заполнения;
* **Родительский элемент** заполняется если для всех новых создаваемых **Объектов** он
  будет один и тот же, его можно
  отредактировать при создании по шаблону;
* **Тип оборудования** - это классификатор объектов (оборудования) по признаку наличия адреса;
* В поле **Название** можете ввести наименование оборудования, его можно будет отредактировать при
  создании по
  шаблону. Если название не заполнено, то автоматически в при создании объекта будет подставлено наименование
  шаблона;
* Флажок **Мобильное оборудование** устанавливается, если при создании **Заявки** вам
  нужно менять адрес объекта;
* Заполните **Участок**, к которому должно принадлежать оборудование (**Участков** может
  быть несколько). Подробнее
  читайте в статье: [Настройка участков](https://wiki.hubex.ru/docs/FAQ/RU/admin/Places.html.md);
* **График работы оборудования** - это справочный параметр, необязательный для заполнения;
* **Класс оборудования** - характеристика оборудования. Если вы не ведете учет по
  **Классам** укажите вариант **По умолчанию**.
  В мобильном приложении без выбора Класса объект создать нельзя. Подробнее читайте в статье: [Классы оборудования](https://wiki.hubex.ru/docs/FAQ/RU/admin/ObjectClass.html.md).

Подробнее о значение и правилах заполнения полей вы можете прочитать в статье [Создание обслуживаемого оборудования](https://wiki.hubex.ru/docs/FAQ/RU/user/CreatingObjects.html.md).

![](https://wiki.hubex.ru/attachments/images/FAQ/USER/CreatingObjTemplates/ObjectTemplate.jpg)

Рассмотрим заполнение полей вкладки **Обслуживание**:

* Укажите **Виды работ**, которые планируете проводить на объектах, созданных по шаблону;
* В поле **На гарантии до** можно внести информацию о гарантийном сроке;
* В поле **Ответственный за объект** можно указать, например, исполнителя, который по плану должен
  обслуживать объекты.

![](https://wiki.hubex.ru/attachments/images/FAQ/USER/CreatingObjTemplates/ObjectTemplate2.jpg)

На вкладке **Дополнительные поля** вы можете указать и заполнить свои персональные поля. Подробнее
читайте в статье:
[Создание дополнительных полей для
объекта](https://wiki.hubex.ru/docs/FAQ/RU/user/AdditionalFields.html.md).

Вы можете создавать разные шаблоны объектов: по заказчикам, участкам и т.д.

После заполнения всех полей
сохраните шаблон.

##### Приемка и маркировка объектов (оборудования) через мобильное приложение исполнителя

Основные возможности меню **Приемка и маркировка** в мобильном приложении исполнителя:

* [Приемка без QR-кода](https://wiki.hubex.ru/docs/FAQ/RU/user/CreatingObjTemplates.html.md#1) - создание новых объектов (оборудования) без использования QR-кода по
  шаблону или без.
* [Приемка по QR-коду](https://wiki.hubex.ru/docs/FAQ/RU/user/CreatingObjTemplates.html.md#2) - это создание новых объектов (оборудования) с использованием QR-кода. Для
  приемки вы можете
  использовать существующий
  QR-код (например, уже наклеенный на оборудование и не использованный ранее в
  HubEx) или создать свой в web-приложении, затем распечатать его и приклеить к объекту (оборудованию).
* [Маркировка по QR-коду](https://wiki.hubex.ru/docs/FAQ/RU/user/CreatingObjTemplates.html.md#3) - это привязка QR-кода к уже существующему объекту (оборудованию). Для
  маркировки
  используется существующий
  QR-код (например, уже наклеенный на оборудование и не использованный ранее в
  HubEx).

Рассмотрим каждый из вариантов по шагам.

**Приемка без QR-кода**:

1. Откройте в мобильном приложении исполнителя меню **Приемка и маркировка**.

![](https://wiki.hubex.ru/attachments/images/FAQ/USER/CreatingObjTemplates/OpenMenu.jpg)

2. Нажмите кнопку **Продолжить без маркировки**.

![](https://wiki.hubex.ru/attachments/images/FAQ/USER/CreatingObjTemplates/Nomark.jpg)

3. Далее выберите способ создания нового объекта: вы увидите все созданные в системе шаблоны и сможете
   выбрать один
   из них, либо выбрать вариант создать без шаблона. Выше мы создавали шаблон объекта, поэтому здесь будем
   рассматривать создание объекта по шаблону.
   Выбираем **Шаблон для объектов кофейни Ривьера**. Название оборудования будет соответствовать
   названию
   выбранного шаблона, его нужно изменить и ввести название создаваемого оборудования. Если необходимо, вы можете
   внести изменения в предзаполненные поля и после проверки нажать кнопку **Создать**.

![](https://wiki.hubex.ru/attachments/images/FAQ/USER/CreatingObjTemplates/NewObj.jpg)

**Приемка по QR-коду** возможна в двух вариантах:

1. у вас уже есть QR-код, которые находится на оборудовании и не внесен в HubEx;
2. вы сначала создаете QR-код в web-приложении без привязки к конкретному оборудованию, распечатываете его и
   приклеиваете на
   оборудование, затем маркируете. Как создать свой QR-код читайте в статье: [Создание шаблона заявки, подача
   заявки по QR-коду](https://wiki.hubex.ru/docs/FAQ/RU/user/CreatingTaskTemplates.html.md).

Рассмотрим порядок действий по приемке по QR-коду:

1. Откройте в мобильном приложении исполнителя меню **Приемка и маркировка**.

![](https://wiki.hubex.ru/attachments/images/FAQ/USER/CreatingObjTemplates/OpenMenu.jpg)

2. Наведите рамку сканера на QR-код и в появившемся окне выберите пункт **Новое оборудование**. Затем
   выберите шаблон
   и продолжите создание нового объекта по аналогии с инструкцией для [Приемка без QR-кода (пункт
   3)](https://wiki.hubex.ru/docs/FAQ/RU/user/CreatingObjTemplates.html.md#p3).

![](https://wiki.hubex.ru/attachments/images/FAQ/USER/CreatingObjTemplates/NewObjQR.jpg)

**Маркировка по QR-коду** работает с уже существующим на объекте QR-кодом, который не
использовался ранее в
HubEx:

1. Откройте в мобильном приложении исполнителя меню **Приемка и маркировка**.

![](https://wiki.hubex.ru/attachments/images/FAQ/USER/CreatingObjTemplates/OpenMenu.jpg)

2. Наведите рамку сканера на QR-код и в появившемся окне выберите пункт **Существующее оборудование**
   и затем выберите
   к какому из объектов нужно привязать QR-код.

![](https://wiki.hubex.ru/attachments/images/FAQ/USER/CreatingObjTemplates/MarkQR.jpg)

Обратите внимание! Привязывая QR-код к **Объекту**, или создавая новый **Объект** по
QR-коду, в HubEx автоматически будет
создан шаблон заявки. Шаблон заявки - это образец заявки с заполненными основными полями. Шаблон заявки вы можете
найти в меню **Заявки - Маркировка**. Подробнее читайте в статье: [Создание шаблона заявки, подача
заявки по QR-коду](https://wiki.hubex.ru/docs/FAQ/RU/user/CreatingTaskTemplates.html.md). В дальнейшем вы сможете подавать **Заявки** по такому QR-коду в мобильном
приложении
исполнителя или заказчика.

Готовые QR-стикеры (QR-шильды) можно заказать в web-приложении по кнопке
**QR-стикеры
на вашем оборудовании**: вы получите уникальные стикеры с вашим логотипом для маркировки обслуживаемого оборудования.

![](https://wiki.hubex.ru/attachments/images/FAQ/USER/CreatingObjTemplates/QRButton.jpg)

В открывшемся диалоговом окне нажмите кнопку **Заказать QR-стикеры**. Запрос будет автоматически
направлен в службу поддержки HubEx и наши специалисты свяжутся с вами в ближайшее время для уточнения информации.

![](https://wiki.hubex.ru/attachments/images/FAQ/USER/CreatingObjTemplates/QRButton2.jpg)

![](https://wiki.hubex.ru/attachments/images/FAQ/USER/CreatingObjTemplates/QRButton3.jpg)

С помощью цифровых шильд вы сможете предоставить своим клиентам удобный способ быстро подать **Заявку** на обслуживание: им нужно будет только отсканировать QR-стикер камерой смартфона. Подробнее об этом читайте в следующей статье: [Создание шаблона заявки, подача заявки по QR-коду](https://wiki.hubex.ru/docs/FAQ/RU/user/CreatingTaskTemplates.html.md).

![](https://wiki.hubex.ru/attachments/images/FAQ/USER/CreatingObjTemplates/QR.jpg)

##### Публикация объекта (оборудования) после маркировки

Для того, чтобы по созданным в мобильном приложении **Объектам** можно было создавать
**Заявки**, необходимо их опубликовать.
Для публикации **Объекта** в web-приложении перейдите в
меню **Объекты|Оборудование** и с помощью фильтров отберите неопубликованные объекты. Неопубликованные
объекты
отмечаются маркером оранжевого цвета.

В контекстном меню (кнопка "три точки") выберите пункт **Опубликовать**.

![](https://wiki.hubex.ru/attachments/images/FAQ/USER/CreatingObjTemplates/Public.jpg)
